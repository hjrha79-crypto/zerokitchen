index.html 버그 4개 수정해줘.

1. "추가" 키워드 세션 합산 버그
"토마토소스 8봉지 추가" 입력 시 _sessionInputs의 이전값과 합산되어 입고 처리되는 버그.
"추가" 키워드 감지 시 _sessionInputs 합산 로직 완전히 건너뛰고 입력값만 바로 inventory_add로 처리.
예: 세션에 토마토소스 11봉 있어도 "토마토소스 8봉 추가" → +8봉만 입고.

2. "육수 네 통" → "육수네" 파싱 오류
한글 숫자(하나/둘/셋/넷/다섯 등)가 품목명 끝에 붙어있을 때 분리가 안 되는 버그.
예: "육수네통" → 품목: 육수, 수량: 4, 단위: 통
_ruleTokenize에서 품목명 끝에 붙은 한글 숫자를 분리하는 로직 강화.
단, "마스카포네" 같은 품목명은 분리되면 안 됨 (2글자 이상 수사만 처리).

3. 설정 변경 명령이 새 품목으로 생성되는 버그
"칠리소스 거래처 쿠치나 푸드로 바꿔 줘" 입력 시 설정 변경 카드가 뜨지 않고
"칠리소스 거래처 쿠치나 푸드로 바꿔 줘"라는 새 품목이 생성됨.
UPDATE_SETTING intent 감지 패턴 확장:
- "X 거래처 Y로 바꿔줘/변경해줘/바꿔줘" 패턴 감지
- "X 최소수량 N으로 바꿔줘" 패턴 감지
- "X 목표수량 N으로 바꿔줘" 패턴 감지
감지 시 설정 변경 카드 표시.

4. 음성 오타로 긴 문장이 단일 품목으로 생성되는 버그
품목명이 10글자를 초과하면 유사 품목 선택 카드에 경고 표시.
카드 문구: "입력이 너무 깁니다. 다시 말씀해 주세요."
[다시 입력] [그대로 추가] 버튼 표시.
